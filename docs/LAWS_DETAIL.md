# DevOps Platform 机器人协作律法全书 (Detailed Laws)

> **定位**：本文档是 `AGENTS.md` 简写指令的详细解释版。当 Agent 对规则边界存在疑问时，必须以此为最高准则。

## 1. 生命周期律 (Lifecycle)

### 1.1 执行态强解锁 (Execution Lock)
- **背景**: 防止 AI 在探讨阶段擅自修改文件，造成意外的代码腐化。
- **边界**: 只要用户的回复中带有“？”或表达了犹豫（如“让我想想”、“你觉得呢”），严禁调用任何写文件工具。
- **DoD**: 必须在回复末尾询问：“方案已就绪，是否授权开始物理编码？”

### 1.2 中断恢复嗅探 (Interruption Recovery)
- **背景**: 开发者经常新开会话，AI 往往不知道前任留下了哪些“半成品”。
- **动作**: 启动会话后的第一步必须是 `git status -u`。
- **冲突处理**: 若发现未跟踪文件，必须假设发生了崩溃，先同步进度再继续工作。

## 2. 架构契约 (Contracts)

### 2.1 幂等性与重放安全 (Idempotency)
- **核心**: 确保在同步任务中断重启后，数据不会重复。
- **实战**: 任何 `INSERT` 操作必须配合 `ON CONFLICT` 或 `exists` 检查。
- **测试强制**: 必须编写一个单元测试，连续运行两次 `worker.run()`，断言数据库记录数不变。

### 2.2 错误谱系强契约 (Error Pedigree)
- **核心**: 拒绝“吞掉异常”。
- **规范**: 严禁 `try: ... except: pass`。所有的 Exception 必须包装成 `AppException(code=xxx)`。
- **取证**: 交付报告中必须注明：本次修改涉及哪些潜在失败点，对应的错误码是多少。

## 3. 工程严谨性 (Engineering Rigor)

### 3.1 宿主机环境红线 (Host Environment)
- **关键**: 解决 Windows PowerShell 编码导致的代码损坏问题。
- **禁令**: 严禁在 PS 中使用 `>` 或 `>>`。这会导致 UTF-8 变成带 BOM 的乱码，破坏 Python 语法。
- **替代**: 所有的文本处理必须调用 `python scripts/utils.py` 进行。

### 3.2 意图锚点 (ADR)
- **背景**: 防止 AI 觉得 lambda 关联很丑，顺手重构成字符串关联，导致测试再次崩溃。
- **约束**: 每次重构前必须扫描 `docs/adr/`。如果某段代码写得很“奇怪”，大概率是为了规避某个底层坑。
