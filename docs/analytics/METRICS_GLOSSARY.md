# 效能度量指标与口径说明 (Metrics Glossary)

本平台内置了多维度分析体系，从**效能、质量、协作、价值、财务**五个维度量化研发过程。

## 1. 研发效能 DORA 指标 (DORA Metrics)

- **部署频率 (DF)**: 成功部署到生产环境的频率。
- **变更前置时间 (LT)**: 代码从提交到部署至生产环境的时长（结合日历扣除节假日）。
- **变更失败率 (CFR)**: 导致失败/回滚的部署占比。
- **平均恢复时间 (MTTR)**: 发生故障后恢复服务所需的平均时间。

## 2. 个人综合贡献度 (User Impact Score)

- **公式**: `Score = (MRs × 10) + (Issues × 2) + (Commits × 1)`。
- **维度**:
  - **协作贡献**: 以 MR 合并数为核心产出权重。
  - **问题管理**: 解决与发现 Issue 的能力。
  - **代码产出**: 基础 Commit 与代码行数。

## 3. 流动效能 (Flow Efficiency) 🌟

- **定义**: `ActiveTime / TotalTime`。
- **逻辑**: 追踪任务从创建到交付的全生命周期，识别流程中的“等待黑盒”。

## 4. 财务与 ROI 效能 (FinOps)

- **资本化率 (CAPEX%)**: 功能开发 (Feature/Epic) 与维护修复 (Bug/Refactor) 的投入占比。
- **ROI Score**: `(业务收益) / (累计投入的人力与基建成本)`。
- **技术债务财务化**: 将 SonarQube 的技术债务转换为财务成本金额，用财务语言量化风险。

## 5. 团队协作与架构健康

- **评审民主度**: 统计 MR 的独立评审人多样性，识别“一言堂”风险。
- **架构脆性指数 (ABI)**: 识别“高耦合+高变动+低质量”构成的技术黑洞。ABI 过高意味着系统急需重构。
- **团队倦怠雷达 (Burnout Radar)**: 通过 WLB 指数（非工作时间提交占比）预警人才流失风险。

## 6. 状态分类定义 (Status Definition)

- **活跃阈值**:
  - **Active**: 过去 30 天有活动。
  - **Dormant**: 过去 90 天无活动。
  - **Churned**: 超过 365 天无活动。
