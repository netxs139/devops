# Traceability Radar API 契约 (Traceability API Contract)

> **API 状态**: 🟢 Active (v1.0)
> **Base URL**: `/api/traceability`

## 1. 聚合雷达数据 (Aggregate Radar)

- **Endpoint**: `GET /radar`
- **Params**:
  - `project_id` (Optional): 产品 ID
  - `days` (Optional): 统计周期，默认 30
- **Response**: 返回 VSM、协作、安全及 ELOC 的聚合指标（详见 `schemas_traceability.py:RadarResponse`）。

## 2. 详情下钻接口 (Detail Drill-down) [NEW]

- **Endpoint**: `GET /detail`
- **Params**:
  - `metric_type`: **必填**。下钻维度类型：
    - `RUBBER_STAMP`: 获取秒批 MR 列表。
    - `VSM_WAITING`: 获取长等待 MR 列表。
    - `HIGH_ELOC`: 获取高贡献 Commit 列表。
    - `VULNERABILITY`: 获取活跃漏洞列表。
  - `project_id` (Optional): 产品 ID。
  - `days` (Optional): 统计周期。
  - `limit`: 分页限制，默认 50。
- **Response**: 返回标准化的列表数据。
- **Response Schema**:
  ```json
  {
    "type": "RUBBER_STAMP",
    "items": [
      {
        "id": "MR IID",
        "title": "标题",
        "author": "作者",
        "value": "具体指标值 (如 2min)",
        "url": "GitLab 链接",
        "timestamp": "发生时间"
      }
    ]
  }
  ```
